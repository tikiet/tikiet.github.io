HTTP essentials for Android: URI vs. URL

一个句话总结：URI 是 URL 的超集

URI 的定义：scheme-name : hierachical-part ? query # fragment, 其中的 hierarchical-part 又可以分为两类：

  • 以 // 开头，包括 authority 和 path 两部分
  • 不以 // 开头，只包括 path

scheme-name其实也和域名一样需要在 IANA 注册。注册的 URI scheme 中有 tel, mailto 等常见的类型，也有things, skype 这样和具体 App 关联的 scheme。

URI 还可以细分成 Absolute URI 和 Relative URI，其中 Relative URI 不包含 scheme-name。

URL 的定义：scheme://user:password@domain:port/path?query#fragment。
URL 相比 URI 增加了从何处获取资源的位置信息。

除 URL，URI 中还包括 URN，一种通过命名空间来表示资源（而非位置）的表示法。
URN 的定义：urn: namespance-identifier : namespance-specific-string。
