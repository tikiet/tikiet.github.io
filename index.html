Android HTTP 摘要: URI 与 URL

一个句话总结：URI 是 URL 的超集

URI (Uniform Resource Identifier)

URI 的定义：scheme-name : hierachical-part ? query # fragment

hierarchical-part 可以分为两类：

  • 以 // 开头，包括 authority 和 path 两部分, 对应的 URI 在 Android 文档中称之为 Hierachical URI
  • 不以 // 开头，只包括 path, 对应的 URI 在 Android 文档中称之为 Opaque URI

scheme-name 其实也和域名一样需要在 IANA 注册。注册的 URI scheme 中有 tel, mailto 等常见的类型，也有things, skype 这样和具体 App 关联的 scheme。

URI 还可以细分成 Absolute URI 和 Relative URI, Relative URI 不包含 scheme-name。

例子：

  • http://www.miui.com: Absolute, Hierarchical
  • mailto:bd-huangye@xiaomi.com: Absolute, Opaque
  • tmp.txt: Relative

URL^2 (Uniform Resource Locator)

URL 的定义：scheme://user:password@domain:port/path?query#fragment。

URL 相比 URI 增加了从何处获取资源的位置信息，这也是其中 Locator 的本意。

http://miui.com 是 URL，而 miui.com 就不是了，因为没有指定 scheme，mailto:hr@xiaomi.com 也不是 URL，因为在其中并不包括具体的资源位置信息，但它是一个合法的 URI。

URN^3 (Uniform Resource Name)

除 URL，URI 中还包括 URN，一种通过命名空间来表示资源（而非位置）的表示法。
URN 的定义：urn: namespance-identifier : namespance-specific-string。

从维基百科拿来的例子：
urn:isbn:0451450523: 《The Last Unicorn》这本书的资源名称

《make it stick: The Science of Successful Learning》读书笔记

Memory is the mother of all wisdom

概要

这本出版于 2014 年的小册子目前还没有中文版，我在 Kindle 商店中购买了电子版后转成 iBooks 可以识别的 epub 格式，之后花了一段时间读完，以下是根据书中内容做的概要。

这本书主要列举了一些当前流行的学习模式、记忆方式的误区，还顺便否定了目前尚未在国内大规模流行开来的“学习类型（learning type）”的概念。这本书批判现在常见的学习方式基本都是徒劳的，就算是主业就是学习的大学生使用的学习方法也是效率低下。与此同时，这些可以追溯到 125 年之前的关于学习方法的研究近年来获得了颇为显著的成果，一些高效率的学习策略颇见端倪，但是这其中有个问题：高效的学习方法不一定符合人的直觉。

以下就是全书的摘要，书中谈到的方法具体有效与否还得根据你自己的实践做出判断。

Learning is misunderstood

通过飞行员 Matt Brown 在驾驶 Cessna 飞机发生事故时的处置方式，引出全书中讨论的“学习”的定义：获得知识和技能，并使之在未来碰到问题或者机会时随时可以从记忆中重新取出来使用。
学习也具有一些不变的特性：

  1. 学习需要记忆，这样才能使现在学到的东西供以后所需。
  2. 人的一生需要不断学习：让自己顺利毕业需要学习、干好工作需要学习技能和与同事合作、退休之后培养新的兴趣也要学习。
  3. “学习”是一项需要习得的技能，同时，最高效的一些策略反倒可能违背直觉。

贯穿全书的关键点

  • 需要耗费精力的学习比不费力气的更加深刻、持久
  • 对于自己学得好还是不好，其实我们自己的估计完全不靠谱：通常我们觉得学习的东西很难，学得很慢的时候会倾向于寻找我们觉得轻松加愉快的学习方式，但是并不知道这样带来的只是稍纵即逝的效果
  • 反复重读课文和题海战术纯粹是浪费时间：这两个应该是目前使用最广泛的学习方法了，但是很遗憾，他们也是效率最低的方法。通常短时间快速、大量的练习会给人一种虚假的“掌握”的感觉，但是真正的掌握需要的是长期记忆，不是这种隔天就忘的学习方法
  • 回忆（retrieval practice）比重读课文要简单高效： 最简单的回忆方法是单词卡片（Flashcard），读完课文或者上完一堂课之后做一个简单的小测验（quiz）也能带来比重读课文更好的记忆效果。虽然大脑不象是可以通过锻炼来变强壮的肌肉，但是构成记忆的神经回路（neural pathway）却的确可以在回顾和练习之后变得强壮。
  • 练习之间间隔一段时间或是将不同类型的练习交叉起来会让人感觉更加生疏，但是这样会让你被迫花更大的力气来回忆，从而导致更加持久的记忆
  • 在教授解法之前先尝试解决问题，虽然会产生更多的错误，但是会有更好的学习效果。
  • 现在流行的学习型的概念没有切实的研究作为支撑
  • 善于总结出区分不同问题本质的人，在碰到陌生问题的时候更能找出正确的解决方法。这种技能可以通过 interleaved practice 和 varied practice 获得。比如交叉做不同类型几何物体体积计算的话，你在碰到一个陌生的物体的时候更有可能找到计算的方法。
  • 我们对于自己知道什么和能做什么的判断经常会受到各种假象的影响，而测试则能保证我们可以校准我们的判断。几乎在任何领域的学习中，你都可以通过测试来检查自己的盲点，让自己更好地掌握相应的技能。
  • 学习新的知识需要以之前的知识作为基础
  • 复述（elaboration）可以让你更好地掌握新的知识：将学习到的新知识和旧有的知识进行连接，并通过你自己的话来复述。新旧知识的关联越多，对其的记忆也就越牢固
  • 将新知识放到更大的语境中有益于学习，比如碰到一个故事的时候将它与你对人类意志和命运不公的理解联系在一起。

Real world HTTP for Android

URL 与 URI

URI 是 URL 的超集。
比如这四个例子：

  1. http://bbs.miui.com
  2. tel:18612345678
  3. mailto:hr@xiaomi.com
  4. file:///etc/hosts

1～4 都属于 URI，但是只有 1 和 4 属于 URL。

URI 的定义：<scheme name> : <hierarchical part> [ ? <query> ] [ # <fragment> ]
其中 <hierarchical part> 如果以 “//“ 开头，则其包括 authority 和 path 两部分，反之则只包含 path。

  • URI: Uniform Resource Identifier
  • URL: Uniform Resource Locator
  • URN: Uniform Resource Name
